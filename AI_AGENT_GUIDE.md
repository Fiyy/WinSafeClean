# AI Agent 协作指南

本文件是给后续 AI Agent 的工作约束。执行任务前必须先读：

1. `README.md`
2. `PROJECT_PRINCIPLES.md`
3. `GOALS.md`
4. `TDD_STRATEGY.md`
5. `docs/RISK_MODEL.md`
6. `PROGRESS.md`

## 工作方式

- 先理解现有文档和测试，再修改代码。
- 不做无关重构。
- 不删除用户未要求删除的文件。
- 不把当前机器状态当作通用事实。
- 不引入需要管理员权限才能运行的默认流程。
- 不引入商业闭源依赖作为核心能力。
- 所有高风险行为必须先有测试和文档。
- 对于可以拆分的单个任务，尽可能使用子 Agent 分拆处理，减少主控对话 context 消耗。

## 修改流程

每个任务遵守：

1. 明确任务目标。
2. 更新或添加测试。
3. 实现最小变更。
4. 运行相关测试。
5. 更新 `PROGRESS.md`。
6. 如果产生架构决策，新增 `docs/adr/NNNN-title.md`。

## 风险用语

报告中不要使用绝对承诺，例如：

- 一定安全
- 绝不会影响系统
- 可以放心删除

应使用可解释的风险表达：

- 低风险：证据显示它是缓存或临时文件，且无活跃引用。
- 中风险：可能是应用数据，建议隔离而不是删除。
- 高风险：存在程序、服务、任务或系统引用，不建议操作。
- 禁止：属于系统关键区域或已知不可手动清理位置。

## 删除能力约束

任何真实删除能力必须满足：

- 有 dry-run 模式
- 有风险等级
- 有证据列表
- 有用户确认
- 有日志
- 尽可能有隔离恢复
- 有测试覆盖

MVP 阶段不得实现真实删除。
