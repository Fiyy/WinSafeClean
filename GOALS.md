# 项目目标

## 问题背景

Windows 长期使用后，C 盘和其他盘会不断增长。用户往往不知道增长来自哪里，也不清楚哪些文件是缓存、日志、残留、安装包、应用数据还是系统依赖。

商业清理工具常见问题：

- 规则不透明
- 风险解释不足
- 过度营销
- 自动删除过于激进
- 付费功能锁定

本项目希望构建一个开源、透明、保守、可解释的替代方案。

## 总目标

构建一个 Windows 安全清理辅助工具，帮助用户理解磁盘占用并做出低风险清理决策。

## MVP 目标

第一版只做只读扫描和报告，不做真实删除：

- 扫描指定磁盘或目录的空间占用
- 找出大文件、大目录和增长风险位置
- 初步判断文件或目录归属
- 建立文件与程序、服务、计划任务、启动项的关系证据
- 输出风险等级和建议动作
- 生成机器可读 JSON 报告和人类可读 Markdown 报告

## 非目标

MVP 不做：

- 自动删除系统文件
- 注册表清理
- 驱动清理
- WinSxS 手动清理
- Windows Installer 缓存删除
- 绕过权限删除文件
- 破解、卸载或禁用商业软件授权组件

## 成功标准

MVP 成功的标准：

- 能解释前 80% 大体积目录的可能用途
- 能明确标记禁止删除区域
- 对高风险目录不给出删除建议
- 所有风险判断都有证据列表
- 核心风险评分逻辑有单元测试
- 所有清理计划在执行前可预览
