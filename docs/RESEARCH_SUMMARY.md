# 调研摘要

日期：2026-05-05

## 结论

目前没有找到一个成熟开源工具能完整满足：

1. 解释文件用途
2. 找出文件和程序、系统组件之间的关系
3. 判断删除是否会影响系统
4. 以保守方式执行或建议清理

但已有多个成熟开源项目可作为参考或集成对象。

## 可参考项目

### BleachBit

用途：开源系统和隐私清理器，支持 Windows，支持预览和规则式清理。

可借鉴：

- 清理规则结构
- dry-run 预览
- 可扩展 cleaner 定义

不足：

- 对任意文件归属解释不足
- 不提供完整依赖关系图

### Bulk Crap Uninstaller

用途：开源卸载器，适合批量卸载和残留清理。

可借鉴：

- 软件识别
- 卸载残留扫描
- 安装程序元数据使用方式

不足：

- 重点是卸载，不是全盘解释和清理

### WinDirStat

用途：磁盘空间可视化分析。

可借鉴：

- 大文件和目录定位
- 用户理解磁盘占用的交互方式

不足：

- 不判断删除风险

### Czkawka / dupeGuru

用途：重复文件、相似文件、空目录查找。

可借鉴：

- 哈希查重
- 结果分组
- 多平台扫描性能

不足：

- 重复文件不代表可安全删除

### Seatbelt

用途：Windows 系统枚举和安全审计。

可借鉴：

- 服务、计划任务、启动项、进程等枚举思路
- 证据收集范围

不足：

- 不是面向普通用户的清理工具

### Dependencies

用途：PE 文件依赖分析。

可借鉴：

- EXE/DLL 依赖树分析
- 模块关系解释

不足：

- 只覆盖可执行文件依赖

## 系统工具基线

Windows 系统组件清理应优先使用官方能力：

- Storage Sense
- Disk Cleanup / cleanmgr
- DISM StartComponentCleanup

不要手动删除 Windows Installer cache、WinSxS、DriverStore 等系统目录。
