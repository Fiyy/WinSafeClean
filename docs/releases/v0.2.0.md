# WinSafeClean v0.2.0

## Date

2026-05-10

## Summary

- Post-release enhancement candidate focused on safer read-only UI workflows and broader Windows ownership evidence.
- WPF Read-Only Ops can now run `scan`, `plan`, and `preflight` safely and load JSON outputs back into the UI.
- Scan reports are easier to triage with largest-item summaries, top-directory summaries, optional directory size calculation, and display-only space-use hints.
- Windows evidence coverage now includes PATH, shortcuts, file associations, and Microsoft Store package ownership.

## Safety Boundary

- `scan`, `plan`, and `preflight` are read-only.
- `quarantine` and `restore` move files only with `--manual-confirmation` and `--i-understand-this-moves-files`.
- `delete` and `clean` are not available.
- Directory quarantine and directory restore are not supported.
- WPF UI only runs `scan`, `plan`, and `preflight`; it rejects cleanup, deletion, quarantine, and restore commands.
- Publishing builds, archives, and hashes files only; it does not run cleanup commands or request elevation.

## Added

- WPF Read-Only Ops runner for safe `scan`, `plan`, and `preflight` execution with JSON auto-load.
- WPF Read-Only Ops options for format, privacy, output files, recursion, item limits, CleanerML, manual confirmation, and directory size calculation.
- Opt-in CLI `--directory-sizes` for `scan` and `plan`.
- Scan report Largest Items and Top Directories summaries.
- WPF display-only space-use hints for common protected, temp, cache, log, crash dump, and download path patterns.
- PATH environment evidence provider.
- Windows shortcut evidence provider.
- Windows file association evidence provider.
- Microsoft Store package ownership evidence provider.
- ADRs 0033 through 0040 covering the new UI, scan, evidence, and safety decisions.

## Changed

- Cleanup plans now treat PATH, shortcut, file association, and Microsoft Store package evidence as conservative reference or ownership evidence that keeps affected items out of quarantine review.
- Windows evidence provider factory now includes the new read-only ownership/reference providers by default.
- Product version metadata updated to `0.2.0`.

## Fixed

- WPF scan report ordering now prioritizes larger items.
- Directory size calculation preserves cautious reasons when partial traversal fails.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI does not store scan history.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
- Sensitive reports should still be generated with `--privacy redacted` before sharing.
