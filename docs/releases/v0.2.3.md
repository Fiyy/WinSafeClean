# WinSafeClean v0.2.3

## Date

2026-05-14

## Summary

- UI workflow patch release focused on making selected results easier to act on safely.
- Adds WPF disposition guidance for Scan Report and Cleanup Plan details.
- Keeps the same safety boundary: the UI explains next steps, but does not delete files or run file-moving commands.

## Safety Boundary

- `scan`, `plan`, and `preflight` remain read-only.
- WPF result disposition guidance is display-only; it does not change risk levels, plan actions, evidence, or cleanup state.
- WPF `Prepare Preflight` is enabled only for `ReviewForQuarantine` plan items with quarantine and restore metadata preview paths.
- WPF UI can build and copy guarded `quarantine` / `restore` CLI handoff commands, but it does not run them.
- `quarantine` and `restore` remain CLI file-moving commands requiring `--manual-confirmation` and `--i-understand-this-moves-files`.
- No direct deletion command or UI delete button is available.

## Added

- `ResultDispositionAdvisor` for selected Scan Report and Cleanup Plan rows.
- Scan Report detail disposition text explaining whether an item should be kept, reviewed in Plan, or reviewed before action.
- Cleanup Plan detail disposition text explaining `Keep`, `ReportOnly`, and `ReviewForQuarantine` next steps.
- ADR 0049 covering the WPF result disposition guidance boundary.

## Changed

- `Prepare Preflight` is disabled for plan rows that are not valid quarantine candidates.
- Product version metadata updated to `0.2.3`.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI recent-document history stores local file paths unless cleared.
- WPF UI builds guarded `quarantine` / `restore` handoff commands but does not execute them.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
