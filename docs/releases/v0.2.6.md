# WinSafeClean v0.2.6

## Date

2026-05-15

## Summary

- UI workflow patch release focused on local run traceability.
- Adds a WPF Run History tab for read-only `scan`, `plan`, and `preflight` runs started from the UI.
- Lets users reopen successful JSON outputs from previous UI runs.

## Safety Boundary

- `scan`, `plan`, and `preflight` remain read-only.
- Run History stores only local metadata: operation kind, target/input path, output path, format, exit code, and timestamps.
- Run History does not store report contents, stdout, stderr, evidence bodies, or command output.
- WPF UI can build and copy guarded `quarantine` / `restore` CLI handoff commands, but it does not run them.
- No direct deletion command or UI delete button is available.

## Added

- `ReadOnlyRunHistory` model and local JSON store.
- WPF `Run History` tab with refresh, open successful JSON output, and clear actions.
- ADR 0052 covering the local read-only run history privacy and safety boundary.

## Changed

- WPF UI records UI-run Evidence Scan, Cleanup Plan, and Safety Check outcomes after the read-only command returns.
- WPF screenshot smoke now captures the Run History tab.
- Product version metadata updated to `0.2.6`.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI recent-document history and Run History store local paths unless cleared.
- Run History does not compare scan results or track quarantine/restore operations.
- WPF UI builds guarded `quarantine` / `restore` handoff commands but does not execute them.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
