# WinSafeClean v0.2.4

## Date

2026-05-14

## Summary

- UI workflow patch release focused on reducing first-run path selection friction.
- Adds WPF Quick Start scan targets for common user-owned locations.
- Keeps the same safety boundary: Quick Start fills paths only and does not run cleanup commands.

## Safety Boundary

- `scan`, `plan`, and `preflight` remain read-only.
- WPF Quick Start only fills the Scan target path and suggests a report output path.
- Quick Start target generation skips empty, duplicate, and protected Windows paths.
- WPF UI can build and copy guarded `quarantine` / `restore` CLI handoff commands, but it does not run them.
- `quarantine` and `restore` remain CLI file-moving commands requiring `--manual-confirmation` and `--i-understand-this-moves-files`.
- No direct deletion command or UI delete button is available.

## Added

- `QuickScanTargetProvider` for Downloads, Desktop, user Temp, and Local AppData scan targets.
- WPF Read-Only Ops Quick Start buttons that populate Scan target and report output paths.
- ADR 0050 covering the Quick Start scan target safety boundary.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI recent-document history stores local file paths unless cleared.
- WPF UI builds guarded `quarantine` / `restore` handoff commands but does not execute them.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
