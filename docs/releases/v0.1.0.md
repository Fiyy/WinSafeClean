# WinSafeClean v0.1.0

## Date

2026-05-08

## Summary

- First local release candidate for the safety-first Windows cleanup assistant.
- Includes read-only scan, plan, and preflight workflows.
- Includes guarded file-level quarantine and restore commands with double confirmation.
- Includes WPF MVP shell for loading existing JSON reports, cleanup plans, and preflight checklists.
- Includes local publish, ZIP archive, checksum manifest, and version metadata baseline.

## Safety Boundary

- `scan`, `plan`, and `preflight` are read-only.
- `quarantine` and `restore` move files only with `--manual-confirmation` and `--i-understand-this-moves-files`.
- `delete` and `clean` are not available.
- Directory quarantine and directory restore are not supported.
- WPF UI opens existing JSON reports/plans/checklists and builds read-only command text; it does not execute cleanup commands.
- Publishing builds, archives, and hashes files only; it does not run cleanup commands or request elevation.

## Added

- Core scan report model with JSON and Markdown output.
- Risk classification for protected Windows paths and unknown evidence.
- Windows evidence providers for services, scheduled tasks, startup entries, uninstall registry entries, running processes, and Authenticode signatures.
- CleanerML safe subset parser and known-cleanup-rule evidence provider.
- Cleanup plan generation with conservative actions and read-only quarantine previews.
- Restore metadata, operation log, preflight checklist, guarded file quarantine, and guarded file restore.
- CLI commands: `scan`, `plan`, `preflight`, `quarantine`, `restore`, `--version`.
- WPF UI for scan reports, cleanup plans, preflight checklists, and read-only command construction.
- Local publish script with optional ZIP archives and `SHA256SUMS.txt`.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI does not execute commands.
- WPF UI does not store scan history.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
- Sensitive reports should still be generated with `--privacy redacted` before sharing.
