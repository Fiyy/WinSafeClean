# WinSafeClean v0.2.1

## Date

2026-05-10

## Summary

- Patch release focused on making the WPF Read-Only Ops workflow usable without hand-written paths.
- Adds file/folder pickers, save-file pickers, automatic output path suggestions, and scan-to-plan handoff.
- Adds cleanup-plan-to-preflight preparation for selected quarantine candidates while keeping the UI read-only for execution.

## Safety Boundary

- `scan`, `plan`, and `preflight` are read-only.
- WPF UI still does not run `quarantine`, `restore`, `delete`, or `clean`.
- `quarantine` and `restore` remain CLI-only and require `--manual-confirmation` plus `--i-understand-this-moves-files`.
- No direct deletion command or UI delete button is available.
- Generated preflight restore metadata input files are review inputs; they do not move or remove files.

## Added

- WPF Read-Only Ops file/folder pickers for scan and plan target paths.
- WPF Read-Only Ops file/folder pickers for CleanerML evidence paths.
- WPF Read-Only Ops JSON file pickers for preflight plan and restore metadata inputs.
- WPF Read-Only Ops save-file pickers for scan, plan, and preflight outputs.
- Automatic non-overwriting JSON output path suggestions on the desktop.
- `Next: Plan` action and automatic scan-to-plan option handoff after successful scan.
- `Prepare Preflight` action on cleanup plan items, generating a preflight restore metadata input file and pre-filling Read-Only Ops preflight fields.
- ADR 0041 covering the path picker and workflow-hint safety boundary.

## Changed

- Read-Only Ops labels now identify target, output, CleanerML, plan, metadata, and checklist fields directly.
- Product version metadata updated to `0.2.1`.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI does not store scan history.
- WPF UI does not execute quarantine or restore.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
