# WinSafeClean v0.2.2

## Date

2026-05-14

## Summary

- UI workflow patch release focused on review ergonomics after `v0.2.1`.
- Adds local recent-document history, read-only result filtering, visible-row copy/export, guarded CLI handoff, and privacy sharing advice.
- Adds a WPF screenshot smoke helper and fixes layout issues found during screenshot review.

## Safety Boundary

- `scan`, `plan`, and `preflight` remain read-only.
- WPF UI can build and copy guarded `quarantine` / `restore` CLI handoff commands, but it does not run them.
- `quarantine` and `restore` remain CLI file-moving commands requiring `--manual-confirmation` and `--i-understand-this-moves-files`.
- No direct deletion command or UI delete button is available.
- WPF visible-row copy/export writes only user-requested CSV text; it does not change source files, plans, metadata, or cleanup state.
- WPF screenshot smoke starts the UI and captures screenshots only; it does not run cleanup commands.

## Added

- WPF Read-Only Ops workflow panel for Scan / Plan / Preflight progress and primary next action.
- WPF Scan Report and Cleanup Plan search, risk/action/type filtering, and sorting.
- WPF local recent JSON document history stored as type, path, and timestamp only.
- WPF Copy Visible and Export CSV actions for currently visible Scan / Plan rows.
- WPF Guarded CLI Handoff for building and copying `quarantine` / `restore` commands after double confirmation.
- WPF privacy sharing advice for `full` versus `redacted` scan and plan output.
- `scripts\smoke-wpf-ui.ps1` screenshot smoke for published UI layout review.
- ADRs 0042 through 0048 covering the new UI workflow and safety boundaries.

## Changed

- Top toolbar layout now wraps controls instead of clipping them at common desktop widths.
- Scan and Plan result lists stretch rows and trim long paths to avoid column overlap.
- Open Preflight now reuses the shared checklist loading path and updates recent-document history.
- Product version metadata updated to `0.2.2`.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI recent-document history stores local file paths unless cleared.
- WPF UI builds guarded `quarantine` / `restore` handoff commands but does not execute them.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
