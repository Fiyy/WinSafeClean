# WinSafeClean v0.2.5

## Date

2026-05-15

## Summary

- UI workflow patch release focused on making the first-run review path clearer.
- Renames the WPF `Read-Only Ops` surface to `Guided Review`.
- Adds current-stage guidance for choosing a target, running Evidence Scan, creating Cleanup Plan, running Safety Check, and reviewing guarded handoff.

## Safety Boundary

- `scan`, `plan`, and `preflight` remain read-only.
- Guided Review can run only `scan`, `plan`, and `preflight`.
- WPF UI can build and copy guarded `quarantine` / `restore` CLI handoff commands, but it does not run them.
- `quarantine` and `restore` remain CLI file-moving commands requiring `--manual-confirmation` and `--i-understand-this-moves-files`.
- No direct deletion command or UI delete button is available.

## Added

- Current-stage title, detail, and safety boundary fields in `ReadOnlyWorkflowPresenter`.
- WPF Workflow panel display for the current Guided Review stage.
- ADR 0051 covering the Guided Review stage language and safety boundary.

## Changed

- WPF tab and primary action labels now use user-facing stage language: Evidence Scan, Cleanup Plan, Safety Check, and Guided Review.
- Recent Safety Check documents use the `Safety Check` display label.
- Product version metadata updated to `0.2.5`.

## Known Limitations

- Published output is a local folder and ZIP archives, not an installer.
- No code signing or auto-update flow is included.
- WPF UI recent-document history stores local file paths unless cleared.
- WPF UI builds guarded `quarantine` / `restore` handoff commands but does not execute them.
- Directory quarantine and directory restore are deferred.
- No real deletion command is available.
