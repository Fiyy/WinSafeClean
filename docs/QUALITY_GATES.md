# 质量门禁

## 代码合并前

任何代码变更合并前必须满足：

- 相关测试已添加或更新
- 相关测试通过
- 风险模型未被绕过
- 文档与行为一致
- 没有引入真实删除作为默认行为
- 没有引入需要管理员权限的默认扫描流程

## 安全门禁

涉及清理计划、隔离或删除的变更必须额外满足：

- 支持 dry-run
- 支持操作日志
- 输出风险等级
- 输出证据列表
- 禁止目录有测试覆盖
- 高风险项不能生成 Delete 动作
- Blocked 项不能生成 Delete 或 Quarantine 动作

## 依赖门禁

新增依赖必须满足：

- 开源许可证可接受
- 不引入商业闭源核心依赖
- 不要求联网才能执行本地扫描
- 不上传用户路径、文件名或哈希
- 能在测试中替换或模拟

## 报告门禁

报告输出必须满足：

- 用户能看懂风险原因
- 机器能解析 JSON 字段
- 不使用绝对化安全承诺
- 不把未知项包装成安全项
