# ADR 0001: 项目方向选择

日期：2026-05-05

## 状态

Accepted

## 背景

Windows 清理工具容易在“释放空间”和“避免误删”之间产生冲突。成熟商业工具通常不够透明，成熟开源工具又大多只覆盖某一个侧面，例如规则清理、卸载残留、重复文件查找或磁盘占用分析。

## 决策

本项目不直接复制传统清理器模式，而是构建一个安全解释和风险评估层：

- MVP 默认只读
- 先输出解释和风险报告
- 后续才引入隔离和删除能力
- 核心逻辑以 TDD 方式开发
- Windows 系统组件清理优先建议官方工具

## 理由

误删系统文件、安装缓存、应用配置或用户数据的代价远高于少释放一些空间。项目的核心价值应是降低用户决策风险，而不是最大化删除数量。

## 后果

优点：

- 安全边界清晰
- 更适合 AI Agent 持续维护
- 容易建立测试和回归样例
- 用户能理解每个建议的依据

代价：

- 初期释放空间能力有限
- 需要较多证据收集适配器
- UI 和报告需要强调解释能力
