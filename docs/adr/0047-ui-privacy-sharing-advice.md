# ADR 0047: UI privacy sharing advice

## Status

Accepted

## Context

The CLI supports `--privacy full|redacted`. Full output is useful for local diagnosis because it preserves paths, reasons, blockers, evidence, and quarantine preview details. The same data can expose private folder names, installed applications, user names, and usage patterns when shared outside the local PC.

The WPF UI already exposes privacy mode controls for scan and plan. Users need an inline reminder about the sharing implications before generating reports.

## Decision

The WPF UI shows privacy sharing advice next to the scan and plan privacy selectors.

For `full`, the UI warns that output includes local paths and evidence and recommends `redacted` before sharing outside the PC. For `redacted`, the UI confirms reduced path exposure. The advice is generated by a pure ViewModel helper with tests.

The advice does not change the selected privacy mode, rewrite reports, redact existing loaded reports, or block local workflows. It is a visible safety prompt only.

## Consequences

- Users receive the warning at the point where report privacy is selected.
- The prompt keeps local review ergonomic while reducing accidental oversharing.
- Redaction remains explicit and implemented by the CLI report generation path.
- Exported or copied full-fidelity data can still contain private paths; users remain responsible for choosing redacted output before sharing.
