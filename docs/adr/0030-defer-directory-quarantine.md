# ADR 0030: Phase 3 暂缓目录隔离和目录恢复

日期：2026-05-08

## 状态

Accepted

## 背景

Phase 3 已经具备文件级 `plan -> preflight -> quarantine -> restore` 闭环。下一步需要判断目录隔离和目录恢复是否进入当前阶段。

目录移动的风险明显高于单文件移动：

- 目录可能包含 junction、symlink、mount point 或其他 reparse point。
- 目录可能混合缓存、配置、数据库、插件和运行时文件。
- 大目录移动可能部分失败，恢复时也可能出现半恢复状态。
- 目录大小、子项 ACL、锁定文件和跨卷移动都会扩大回滚复杂度。

## 决策

Phase 3 暂不实现目录隔离和目录恢复。当前执行器继续只支持文件：

- `quarantine` 遇到目录 source 时失败，不移动。
- `restore` 遇到目录 quarantine path 时失败，不移动。
- `plan` 仍可报告目录和生成人工审阅候选，但真实执行必须拒绝目录移动。

未来若实现目录隔离，必须先补充独立设计和测试：

- 目录树枚举快照。
- reparse point、junction、symlink 和 mount point 阻断策略。
- 原子性或可恢复的分步移动日志。
- 子项 hash/大小/ACL 记录。
- 部分失败恢复策略。
- 大目录性能和磁盘空间检查。

## 理由

当前项目的安全目标是先提供一个可解释、可回滚的最小闭环。文件级隔离已经能覆盖低风险缓存文件、日志文件和崩溃转储等常见候选。目录隔离会引入大量额外边界，若在当前阶段仓促实现，会显著增加误移动和不可恢复风险。

## 后果

优点：

- Phase 3 范围保持清晰。
- 执行器失败模式更简单，可测试性更高。
- 避免把复杂目录语义伪装成普通文件移动。

限制：

- 当前无法隔离整个目录。
- 大型缓存目录只能先报告或由用户/系统工具处理。
- 后续若要支持目录，必须新增 ADR 和测试矩阵。
