# ADR 0008: 显式递归扫描策略

日期：2026-05-05

## 状态

Accepted

## 背景

Phase 1 最初保持非递归扫描，以避免大目录扫描、权限噪音、符号链接循环和跨卷遍历。后续 CLI 已具备 `--max-items`、异常降级、报告隐私模式和稳定测试探针，可以在显式开关下提供有限递归扫描。

递归扫描仍必须保持只读，不读取文件内容，不跟随危险跳转，不执行删除、隔离或修复。

## 决策

CLI 支持：

```text
--recursive
--no-recursive
```

默认仍为非递归；`--recursive` 必须由用户显式传入。

递归策略：

- 扫描根目录本身不作为报告项返回，返回其子项和子树。
- 目录项会作为报告项返回，然后再进入子目录。
- `MaxItems` 是整次扫描的全局返回项上限。
- 达到 `MaxItems` 后立即停止追加结果。
- 目录内枚举结果在当前预算内按 `OrdinalIgnoreCase` 排序，保持报告稳定。
- 默认不跟随 reparse point、junction 或 symlink；只报告该路径本身。
- 属性读取失败时保守处理为不跟随。
- 单个目录无法枚举时降级为 `Unknown / ReportOnly`，不让整个扫描崩溃。

## 理由

递归扫描有实际诊断价值，但必须由显式参数触发。全局 `MaxItems` 和不跟随 reparse point 是 Phase 1 的关键安全边界，可以控制扫描成本和避免循环。

## 后果

优点：

- 用户可以发现深层文件和目录。
- 仍然保持只读。
- `MaxItems` 控制整次扫描输出规模。

代价：

- 递归扫描仍可能比单层扫描慢。
- 目录大小仍不做递归汇总。
- 当前没有取消令牌；长扫描的取消机制需要后续单独设计。
- 不跟随 reparse point 意味着部分链接目标不会进入报告。
